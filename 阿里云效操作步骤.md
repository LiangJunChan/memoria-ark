# 阿里云效 CodePipeline 操作步骤（详细图文版）

## 第一步：登录阿里云效

1. 访问 https://devops.aliyun.com
2. 使用阿里云账号登录
3. 首次使用需要同意服务条款

## 第二步：进入流水线页面

1. 登录后，点击左侧菜单 **「流水线」**
2. 或者访问：https://devops.aliyun.com/pipeline

## 第三步：创建新流水线

### 3.1 点击「新建流水线」
- 位置：页面右上角蓝色按钮 **「新建流水线」**

### 3.2 选择流水线模板
- 选择 **「空白模板」**（第一个选项）
- 或者选择 **「移动应用」** 分类下的模板
- 点击 **「创建」**

### 3.3 填写流水线基本信息
- **流水线名称**：`memoria-ark-android`（或您喜欢的名字）
- **流水线描述**：`忆存·数字方舟Android APK自动构建`（可选）
- 点击 **「确定」**

## 第四步：配置代码源（您卡住的地方）

### 4.1 进入代码源配置
- 在流水线编辑页面，找到 **「代码源」** 或 **「源」** 卡片
- 点击 **「添加代码源」** 或 **「编辑」**

### 4.2 选择代码源类型
- 选择 **「GitHub」**（如果您的代码在GitHub）
- 或选择 **「GitLab」**（如果您的代码在GitLab）
- 或选择 **「阿里云Code」**（如果您的代码在阿里云Code）

### 4.3 授权访问
- 点击 **「授权」** 或 **「连接」**
- 会跳转到授权页面，点击 **「授权」**
- 授权成功后返回阿里云效

### 4.4 选择仓库和分支
- **仓库**：选择您的 `memoria-ark` 仓库
- **分支**：选择 `main` 或 `master`
- 点击 **「确定」** 或 **「保存」**

## 第五步：添加构建任务（关键步骤！）

### 5.1 找到构建任务区域
- 在流水线编辑页面，找到 **「构建」** 或 **「任务」** 区域
- 点击 **「添加任务」** 或 **「添加步骤」**

### 5.2 选择任务类型
- 选择 **「构建」** → **「命令行」** 或 **「Shell」**
- 或者选择 **「构建」** → **「自定义」**

### 5.3 粘贴构建脚本

**将下面的完整脚本粘贴到编辑器中：**

```bash
#!/bin/bash
set -e

echo "========== 忆存·数字方舟 Android构建开始 =========="
echo "构建时间: $(date)"

# 安装Java 17
echo "========== 步骤1: 安装Java 17 =========="
apt-get update -qq
apt-get install -y -qq wget apt-transport-https gnupg > /dev/null 2>&1
wget -qO - https://packages.adoptium.net/artifactory/api/gpg/key/public | apt-key add -
echo "deb https://packages.adoptium.net/artifactory/deb $(awk -F= '/^VERSION_CODENAME/{print$2}' /etc/os-release) main" | tee /etc/apt/sources.list.d/adoptium.list
apt-get update -qq
apt-get install -y -qq temurin-17-jdk > /dev/null 2>&1
export JAVA_HOME=/usr/lib/jvm/temurin-17-jdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
echo "Java版本:"
java -version

# 安装Android SDK
echo "========== 步骤2: 安装Android SDK =========="
export JAVA_HOME=/usr/lib/jvm/temurin-17-jdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
export ANDROID_HOME=/opt/android-sdk
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools
mkdir -p $ANDROID_HOME
cd $ANDROID_HOME
echo "下载Android SDK..."
wget -q https://dl.google.com/android/repository/commandlinetools-linux-11076708_latest.zip
echo "解压Android SDK..."
unzip -q commandlinetools-linux-11076708_latest.zip
mkdir -p cmdline-tools/latest
mv cmdline-tools/bin cmdline-tools/lib cmdline-tools/NOTICE.txt cmdline-tools/latest/ 2>/dev/null || true
echo "接受许可证..."
yes | sdkmanager --licenses > /dev/null 2>&1 || true
echo "安装SDK组件..."
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0" > /dev/null 2>&1
echo "Android SDK安装完成"

# 构建前端
echo "========== 步骤3: 构建前端 =========="
cd frontend
npm ci --silent
npm run build
echo "前端构建完成"

# 同步Capacitor
echo "========== 步骤4: 同步Capacitor =========="
npx cap sync android
echo "Capacitor同步完成"

# 构建APK
echo "========== 步骤5: 构建APK =========="
export JAVA_HOME=/usr/lib/jvm/temurin-17-jdk-amd64
export PATH=$JAVA_HOME/bin:$PATH
export ANDROID_HOME=/opt/android-sdk
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools
cd android
./gradlew assembleDebug --no-daemon
echo "APK构建完成"

# 输出APK信息
echo "========== 构建完成 =========="
echo "APK文件:"
ls -lh app/build/outputs/apk/debug/*.apk
echo "构建时间: $(date)"
echo "========== 忆存·数字方舟 Android构建结束 =========="
```

### 5.4 保存配置
- 点击 **「保存」** 或 **「确定」**

## 第六步：保存并运行流水线

### 6.1 保存流水线
- 点击页面右上角的 **「保存」** 按钮
- 流水线上线成功

### 6.2 手动触发构建
- 点击 **「运行」** 或 **「立即执行」** 按钮
- 或者等待代码推送自动触发

### 6.3 查看构建进度
- 在 **「执行记录」** 或 **「运行历史」** 中查看
- 点击具体的构建记录，查看实时日志

### 6.4 下载APK
- 构建成功后，在 **「构建产物」** 或 **「Artifacts」** 中
- 找到 `app-debug.apk` 文件
- 点击 **「下载」** 按钮

## 第七步：自动化配置（可选）

### 7.1 自动触发构建
- 在流水线设置中，启用 **「代码推送自动触发」**
- 每次推送代码到 `main` 分支，自动触发构建

### 7.2 构建通知
- 配置 **「钉钉通知」** 或 **「邮件通知」**
- 构建成功或失败时，自动发送通知

### 7.3 多环境构建
- 配置 **「开发环境」**、**「测试环境」**、**「生产环境」**
- 不同分支触发不同环境的构建

## 常见问题解决

### Q1: 代码源授权失败？
- 确保阿里云账号和GitHub/GitLab账号已绑定
- 重新授权，点击 **「重新授权」** 按钮
- 检查GitHub/GitLab的OAuth授权状态

### Q2: 构建失败，日志显示内存不足？
- 阿里云效默认提供 **2GB内存**，足够构建APK
- 如果失败，尝试在脚本中添加 `export GRADLE_OPTS="-Xmx1536m"`

### Q3: 构建时间太长？
- 首次构建需要下载依赖（约5-10分钟）
- 后续构建会缓存依赖（约2-5分钟）

### Q4: 如何查看详细日志？
- 在构建记录页面，点击 **「查看日志」**
- 可以实时查看构建进度和详细日志

### Q5: APK下载后如何安装？
- 将APK文件传输到Android手机
- 在手机上点击APK文件安装
- 如果提示「未知来源」，在设置中允许安装未知来源应用
